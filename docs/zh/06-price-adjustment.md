# 价格退让机制

## 问题：限价单的成交困境

限价单（LMT）的优势是以指定价格或更好价格成交，缺点是**可能永远无法成交**。

在铁鹰策略中这个问题尤为突出：

- 你提交了一个价格为 -1.25 的限价单
- 市场当前的组合中间价是 -1.30（你需要出价比中间价更优才能吸引对手方）
- 如果你不变价，可能几小时都没有成交
- 如果你频繁改价，又可能改得太快，成交在不理想的价格

**解决方案**：定时阶梯退让 + 预判跳转。

---

## 基本策略：定时阶梯退让

### 核心逻辑

```python
# 伪代码 — 完整的监控循环
while 未成交 and 未超时:
    if 距离上次调整 >= 调整间隔:        # 默认 3 分钟
        if 可以调整(订单状态 == "Submitted"):
            新价格 = 当前价格 + 步长       # 步长 = 最小变动单位 0.05
            modify_order_price(订单ID, 新价格)
    等待 1 秒
```

### 两个关键参数

| 参数 | 说明 | 为什么是这个值 |
|:-----|:-----|:-------------|
| **调整间隔** | 每次退让之间的等待时间 | 太短：频繁改价触发 TWS 限流；太长：可能错过成交窗口 |
| **退让步长** | 每次价格变化的幅度 | 必须是最小变动单位（SPX 期权为 0.05）的整数倍 |

### 最小变动单位对齐

SPX 期权的价格精度为 0.05，所以任何价格必须是 0.05 的整数倍：

```python
import math
initial_price = math.floor(raw_price / 0.05) * 0.05
```

向下取整是为了让价格更有利于成交（在 BUY 方向下，更低的价格更容易成交）。

## 进阶策略：预判跳转

阶梯退让有一个问题：如果你按步长一步步退让，可能在到达"底线价格"时才成交，而这个价格已经超出了你预先设定的可接受范围。

**解决方法**：在退让前先做一步预判。

```python
# 预判逻辑
planned_new_price = current_price + step_size   # 计算下次退让后的价格

if planned_new_price >= FIXED_PRICE_BARRIER:     # 如果触及底线
    # 不经过底线价格，直接跳到特殊价格
    modify_order_price(order_id, SPECIAL_PRICE)
else:
    # 正常退让
    modify_order_price(order_id, planned_new_price)
```

### 为什么预判跳转有效？

```
价格轴（负数表示你付出的权利金越来越少）：
  -1.40  ← 初始价格（最有利于你）
  -1.35  ← 第 1 次退让
  -1.30  ← 第 2 次退让
  -1.25  ← 第 3 次退让
  -1.20  ← 第 4 次退让
  -1.15  ← 第 5 次退让
  -1.10  ← 第 6 次退让
  ↓
  ┌──────┐
  │ -1.05│ ← 底线价格（FIXED_PRICE_BARRIER）
  └──────┘
  ↓
  ┌──────┐
  │ -1.25│ ← 特殊价格（跳过底线，跳到曾经成交过的合理位置）
  └──────┘
```

当预判到下次退让将触及底线时：

1. **不调到 -1.05**（底线价格），因为 -1.05 本身不是一个理想的成交价位
2. **直接跳转到 -1.25**（一个历史成交密集区），在价格更优的区间等待成交
3. 跳转后**停止自动调整**，不再继续退让

这种"一次跳转"的设计避免了"触及底线 → 再退让 → 越退越差"的螺旋。

## 防重复修改

多线程环境下，监控循环和成交回调可能同时触发价格修改。需要一个保护机制：

```python
class OrderModificationGuard:
    def __init__(self, cooldown=180):
        self._busy = set()         # 正在修改中的订单
        self._cooldown = cooldown  # 冷却时间（秒）

    def can_modify(self, order_id, status):
        if order_id in self._busy:
            return False            # 已经有修改在进行中
        if status != "Submitted":
            return False            # 只有已提交的订单可以修改
        if time.time() - last_time < self._cooldown:
            return False            # 冷却未结束
        return True

    def start_modification(self, order_id):
        self._busy.add(order_id)    # 锁定

    def finish_modification(self, order_id):
        self._busy.discard(order_id)  # 解锁
```

三重保护：

1. **忙碌锁**：同一时间一个订单只能有一个修改在途
2. **状态检查**：只有 `Submitted` 状态的订单才需要修改
3. **冷却时间**：避免过频繁的修改请求

## 截止时间管理

订单监控不可能无限持续下去。需要设定一个截止时间：

```python
# 美东时间 16:10 为默认截止
end_time = now.replace(hour=16, minute=10, second=0)

# 如果当前时间已经过了 16:10，顺延到次日
if now > end_time:
    end_time += timedelta(days=1)
```

到达截止时间后：

- 如果订单仍未成交 → 取消订单 → 策略执行失败
- 记录失败原因（"到达截止时间"）
- 可选择切换到备选策略或通知用户

## 以铁鹰策略为例的应用场景

假设铁鹰组合理论中间价为 -1.40，初始价格设为 -1.40（向下取整后）：

```
订单提交时间：15:30 ET
截止时间：16:10 ET（当日）
总可用时间：40 分钟

调整间隔：3 分钟
最大调整次数：~13 次（40 / 3）

15:30  提交订单 @ -1.40
15:33  未成交 → 调整到 -1.35
15:36  未成交 → 调整到 -1.30
15:39  未成交 → 调整到 -1.25
15:42  未成交 → 调整到 -1.20
15:45  【成交！】@ -1.20
       → 停止监控 → 记录成交价
```

如果市场流动性不足，价格一路退让：

```
15:30 → -1.40
15:33 → -1.35
15:36 → -1.30
15:39 → -1.25
15:42 → -1.20
15:45 → -1.15
15:48 → -1.10
15:51 → 预判：-1.05 将触及底线
      → 直接跳到特殊价格
      → 标记"已执行特殊调整"
      → 等待成交（不继续退让）
...
16:10 → 截止时间，未成交，返回失败
```

## 小结

价格退让机制的要诀可以概括为：

1. **定时退让，不急不躁**：3 分钟的间隔既给了市场反应时间，也不会拖太久
2. **预判跳转，避免深陷**：在触及底线前跳到合理价位，而不是一步步踏入陷阱
3. **一次跳转，适可而止**：特殊价格调整只执行一次，之后不再继续退让
4. **截止时间，明确退出**：到了收盘时间坚决离场，不留隔夜风险
